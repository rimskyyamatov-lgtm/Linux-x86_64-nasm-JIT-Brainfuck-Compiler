Brainfuck JIT Compiler

Linux x86_64 NASM

概要

このプログラム bfc.asm は Brainfuck を実行時に x86_64 機械語へ変換する JIT コンパイラです。

  • NASMで実装
  • Linux syscall使用
  • mmapで実行可能メモリ確保
  • Brainfuck → ネイティブコード

GitHub

GitHub Repository

ビルド方法

nasm -f elf64 bfc.asm -o bfc.o
ld bfc.o -o bfc

./bfc program.bf

メモリ構造

section .bss

tape    resb 30000     ; Brainfuckメモリ
program resb 131072    ; ソースコード
stack   resq 8192      ; ループスタック

Brainfuckの標準仕様に合わせて 30000バイトのメモリテープを確保しています。

プログラム開始

global _start

_start:

mov rbx,rsp
mov rax,[rbx]
cmp rax,2
jl exit

ここでは

  • コマンドライン引数数を取得
  • Brainfuckファイルの指定を確認

ファイル読み込み

mov rdi,[rbx+16]
mov rax,2
xor rsi,rsi
xor rdx,rdx
syscall

Linux syscall open を使って Brainfuckソースファイルを開いています。

JITメモリ確保

mov rax,9
xor rdi,rdi
mov rsi,1048576
mov rdx,7
mov r10,34
mov r8,-1
xor r9,r9
syscall

ここでは mmap を使って 実行可能メモリを確保しています。

このメモリに生成した機械語を書き込みます。

Brainfuck → 機械語変換

コンパイラは次の命令を変換します。 

> ポインタ右
< ポインタ左
+ インクリメント
- デクリメント
. 出力
, 入力
[ ループ開始
] ループ終了

例えば + は次の機械語になります。 

mov byte [r14],0x41
mov byte [r14+1],0xfe
mov byte [r14+2],0x45
mov byte [r14+3],0x00

生成コード実行

mov r13,tape
call r15

JITで生成したコードを 直接 call して実行します。
